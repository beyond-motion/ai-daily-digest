## 🧠 Anthropic 再次延期 Fable：OpenAI 凭"确定性"赢走付费用户

**来源**：Simon Willison's Weblog · 2026-07-12 21:20

GPT-5.6 Sol 的发布带来连锁反应——Anthropic 不得不**第三次延期** Claude Fable 5 在付费订阅中的可用窗口。

### 官方声明要点

Anthropic 这次给出明确截止日期："We're extending Claude Fable 5 access on all paid plans, as well as keeping Claude Code's weekly rate limits 50% higher, **through July 19**. As before, you can use up to half of your weekly usage limit on Fable 5. After that, you can continue using Fable 5 with usage credits, or switch to another model to keep working within your remaining limits."

### Anthropic 的一贯说辞

> "Anthropic's original rationale for this was compute constraints - they wanted a better idea of both demand and compute availability before committing to keeping the new model cheap for subscribers."

延一次可以叫"了解需求"，延两次叫"控制供给"，**延三次就只能叫"被竞争对手打了个措手不及"**。

### OpenAI 阵营的姿态

OpenAI 的 Thibault Sottiaux 今天同步发推：
- **临时取消** Plus/Business/Pro 全档位的 5 小时使用上限限制
- 即将推出 GPT-5.6 Sol 的能效优化（"less usage being used so that it can take you further"）
- **过去 48 小时 Codex 与 ChatGPT Work 的活跃用户已经冲到 600 万**，并即将发放使用额度重置

Willison 的判断一针见血："At this point I think Anthropic should change track and keep Fable permanently available on those plans. **OpenAI are winning users simply due to the uncertainty that surrounds Fable access.**"

## 🍎 TwoMillionKit：用 Codex 写出来的"苹果合规拆招器"

**来源**：Daring Fireball（Guilherme Rambo）· 2026-07-12 18:04

macOS 27 这次给第三方应用开放了 Private Cloud Compute，但前提是"App Store 小企业计划 + 累积下载量低于 200 万"。Rambo 的某款 iOS App 早在 10 年前就突破了这个门槛，导致他**所有**应用都无法接入 PCC。Apple 自己的 App Store Connect 报表与"超过 200 万下载"的说法对不上，Rambo 索要原始数据后杳无音信。

### 解决方法：用 GPT-5.6 Sol 反向利用 Apple 的命令行

macOS 27 自带了 `fm` 命令行工具，本意是让用户在终端里就能跑本地模型或 PCC。Rambo 用他刚拿到的 Codex 额度，让 GPT-5.6 Sol 写了一个 Swift 包——**`TwoMillionKit`**：

> "All it does is provide a LanguageModel implementation that uses the fm command-line tool under the hood, meaning that any Mac app can use the Private Cloud Compute model without requiring a special entitlement from Apple."

### 限制（也是命名由来）

- 不支持沙箱化应用——所以 Mac App Store 上架的应用用不了
- 仅限**非** App Store 分发的 Mac 应用开发者

库名"TwoMillion"既是讽刺苹果的 200 万门槛，也是反向致敬——把"被排除在外"变成"我有办法绕过去"。

> "This is a workaround for Apple's current limitation that only grants access to Private Cloud Compute to 'developers in the App Store Small Business Program with fewer than two million first time App Store downloads'. Hence Rambo's clever name for the framework."

一个用 OpenAI 的代币，写出来替自己用 Apple 的能力——上一次这种"用对手的工具拆对手的墙"还是 jailbreak 时代的玩法。

🔗 https://github.com/insidegui/TwoMillionKit

## 👤 DRI（直接负责人）：LLM agent 不应，也不该"背上锅"

**来源**：Simon Willison's Weblog · 2026-07-12 23:57

Simon Willison 翻出"Directly Responsible Individuals"这一概念，**直指当下 AI agent 落地的最大盲区**。

### 什么是 DRI

GitLab Handbook 的解释最清楚："the person who is **ultimately accountable** for the success or failure of a specific project, initiative, or activity"。这个概念起源于 Apple——但 Willison 关注的不是它的历史，而是它**为什么不能套到 agent 身上**：

> "I don't think an agent should *ever* be considered the DRI for a project — that's something that feels uniquely human to me, because humans can take accountability for their actions where machines cannot."

### 1979 年的 IBM 那张老幻灯片

Willison 引用了一张传奇幻灯片里的金句：

> **"A computer can never be held accountable, therefore a computer must never make a management decision."**

——IBM，1979

把这句话和今天所有"AI agent 全自动跑流程"的产品放在一起读，会发现一个尴尬的现实：**当 agent 在 Slack 里直接 @ 同事、调用 CI、甚至自动发邮件时，它正在做的事，比 1979 年 IBM 警告的"管理决策"还要激进**。但今天没有人在问"谁来负责"。

Willison 没有给答案——这本来就该是产品经理、CTO、安全团队共同回答的问题。但他把这条线划得很清楚：**责任只能落在人头上**。

🔗 https://simonwillison.net/2026/Jul/12/directly-responsible-individuals/

## 🖼️ "Every Frame Perfect"：Tonsky 对所有界面设计者的一记闷棍

**来源**：Tonsky · 2026-07-12 (Gruber link blog 收录 19:48)

Wayland 项目的设计目标是"every frame is perfect"。Nikita Prokopov 说：**这不该只是底层渲染的口号，应当是所有 UI 工作者的纪律**。

### 核心信条

> "The rule of thumb is: **If I take a screenshot of your app at any moment, you should be able to explain what I see.**"

### 反例：Safari

Safari 里 placeholder 文本从中间淡出，但光标却**从左侧开始**动画——一个无伤大雅的小错位：

> "It does create a feeling that these two components are not in sync with each other. Next thought: maybe they weren't designed together? If so, then they might not work well together. **That's how trust is lost.**"

### 反例：Photos

切换 Crop 与 Adjust 模式时，图片立即吸附到位，但 crop 边框却开始动画——

> "This creates a false feeling that something subtly changes when you switch between modes. And you know what? **I don't want my UI to give me false feelings.** I want it to be a precise instrument, not an animated toy."

### 反例：YouTube

> "They had the simplest task in the world: move a rectangle from one position to another! Yet they decided to do something very strange."

### 信任的反向逻辑

> "Why care about every frame? It builds trust. Users can't see the code, **so UI is the only way for them to judge the quality of the app**. If UI looks good, that means developers had time to polish it, which means that they probably spent a comparable amount of time to iron out the code."

这道理反过来说更狠——**当你的应用掉帧、白屏、动画跑一半，用户不知道是 GPU 问题、JS 引擎问题、还是你的代码本来就烂，但他们的本能反应都会指向第三条**。Tonsky 的标题政治不正确但诚实："If you cut corners on interstitial states 'because no one will notice', **you'll start cutting corners elsewhere**。"

🔗 https://tonsky.me/blog/every-frame-perfect/

## 💾 Stacks 与 Lunacy：一位开发者用 Swift 把 90 年代的 Mac 体验"装"回 macOS 27

**来源**：Daring Fireball（John Gruber）· 2026-07-12 15:42 与 17:19

Jeff Halter 的 morphing.cloud 这周连发两个项目，**都是用 Swift 把经典 Mac 软件重新搬回 Apple Silicon**。

### Stacks —— HyperCard Player

> "Run HyperCard stacks directly on your modern Mac. **No emulator required!** Browse the Internet Archive's HyperCard collection and run stacks with one-click. Period-accurate typography. Sound, instruments, and MacinTalk speech synthesis. Cross-stack navigation."

Gruber 用了"simultaneously Mac-assed 2026-style and Mac-assed 1987-style"来评价。原生 Swift 应用，无模拟器，提供**时代正确**的字体、声音合成、跨 stack 导航——几乎是给怀旧情绪做了一次 1:1 的数字化修复。

### Lunacy —— Lunatic Fringe 现代化

HyperCard 之外，Halter 还顺手重制了 90 年代的经典屏保+街机游戏 Lunatic Fringe。原版来自 Ben Haller，藏在 Macintosh More After Dark 软件包里——按 Caps Lock 就能进入射击模式，拿高分。

> "Created by Ben Haller and released in the early '90s as part of the Macintosh More After Dark software package... Running Lunatic Fringe on a modern computer has been a challenge. ... **Lunacy brings Lunatic Fringe to the present**: a native Swift app with a built-in emulation engine that runs the original module, unchanged, on modern Apple platforms. ... including CRT simulation to make the game look a lot more like it did back in the day."

### 时代对照

整个 2026 年，Apple 在 SwiftUI 上的"Mac-assed"程度仍然让独立开发者抓狂（参考昨天 Paulo Andrade 的 WWDC 27 更新）。与此同时，Halter 用 Swift 写一个 retro app，**UI 一比一复刻 1987 年的 HyperCard 体验**——这件事本身就是对当下所有"扁平化、磨砂、动效"风潮的一记耳光：**真正难的是把 UI 写"少"**，而不是堆特效。

🔗 https://morphing.cloud/hypercard/ · https://morphing.cloud/lunacy/
