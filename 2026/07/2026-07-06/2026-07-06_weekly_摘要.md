## 🏆 本周头条

### 1. Claude Sonnet 5 发布：性能接近 Opus，新分词器暗涨 30%

Anthropic 在 6/30 发布 Claude Sonnet 5，官方定位"性能接近 Opus 4.8，价格更低"——输入 $3/百万、输出 $15/百万，与 4.6 持平。

**但隐藏的涨价更关键**：Sonnet 5 换了新分词器，同一段文本比 4.6 多产生约 **30% 的 token**。Simon Willison 用 Claude Token Counter 对大文档做了交叉验证。等价于实际价格上涨三成。

其他变化：上下文窗口 **100 万 token**、最大输出 **12.8 万 token**；不再支持 `temperature` / `top_p` / `top_k`（统一自适应 thinking）；网络任务能力远低于 Mythos 5，因此未受出口管制阻断。

🔗 https://simonwillison.net/2026/Jun/30/claude-sonnet-5/

### 3. Meta 正在"摧毁自己的工程组织"

- 追踪所有工程师的**键盘和鼠标点击**

### 4. Safari 引入 MCP Server：AI Agent 可以直接读 DOM

## 🤖 AI 与大模型

### Claude Fable 的实战表现

**sqlite-utils 4.0rc2**：Simon 用 Fable 5 在 iPhone 上 Claude Code 完成了 37 个 prompt / 34 个 commit / 30 个文件的 release review——Fable 自己挖出了一个**导致数据丢失的严重事务 bug**（`delete_where()` 没 commit，后续操作全部回滚）。这是"AI 做真正重要的代码审查"的里程碑。

**llm-coding-agent 0.1a0**：Simon 让 Fable 5 根据 spec 从零实现一个 Claude Code 风格 agent，包括 Simon 没要求的 Python API。Fable 一次写成。

**DSPy 优化 Datasette Agent**：Fable 发现系统提示中"不要调用 describe_table"反而让模型猜测列名、触发错误重试循环——通过结构化评估自动定位优化点。

🔗 https://simonwillison.net/2026/Jul/5/sqlite-utils-fable/

### 更好的模型，更糟的工具调用

Armin Ronacher（Flask 作者）发现一个反直觉现象：**Opus 4.8 和 Sonnet 5 在调用 Pi 的 edit 工具时会瞎编字段名**。编辑内容正确，但 schema 不匹配导致工具拒绝。

他的假说：Anthropic 通过 RL 专门训练新模型对齐 Claude Code 内置 edit 工具（search & replace），殃及所有第三方 harness。与此同时 OpenAI 的 Codex 用 `apply_patch`——模型被训练成忠于各自"主工具"，生态分裂加剧。

🔗 https://simonwillison.net/2026/Jul/4/better-models-worse-tools/
