### 2. 英国 CMA 拟强制开放 App Store 跳转与 NFC

**来源**：Daring Fireball（John Gruber）｜2026-06-30

英国竞争与市场管理局（CMA）启动新一轮咨询，拟要求 Apple 和 Google：

- **允许"跳转（steering）"**：开发者可引导用户前往平台外支付，Apple 当前禁止、Google 当前限制；
- **开放 iOS NFC**：让非 Apple 钱包能用 iPhone 的 NFC 硬件；
- **费用公平合理**：跳转收取的费用需低于现行应用商店佣金，节省的成本须让利给消费者或投入再创新。

CMA 明确把日本的 MSCA（移动软件竞争法）作为参照模板。如果落地，Apple 在英国将面临类似欧盟 DMA 的结构变化，但走的是"量身定制"路径。

Gruber 同时提到一个讽刺点："法律怎么强制平台必须把节省的成本让利给消费者？"——这正是 Apple 一直想反驳却难以正面应对的命题。

🔗 https://www.gov.uk/government/news/cma-consults-on-new-requirements-for-apple-and-googles-mobile-platforms

## 🤖 AI 与编程

### 6. Claude Sonnet 5 发布：新分词器带来隐性涨价

**来源**：Simon Willison｜2026-06-30

Anthropic 早上发布 Claude Sonnet 5，官方称"性能接近 Opus 4.8，价格更低"。系统卡同时披露：Sonnet 5 的网络任务能力远低于 Mythos 5，因此安全保障与 Opus 4.7/4.8 相当——这正是它能不受美国政府出口管制阻断的原因。

API 关键变化：

- **不再支持** `temperature` / `top_p` / `top_k` 三个采样参数（统一改为自适应 thinking）
- **上下文窗口 100 万 token，最大输出 12.8 万 token**
- 平台能力与 Sonnet 4.6 保持一致
- **价格**：输入 $3/百万、输出 $15/百万（与 4.6 相同，至 8/31 折扣 $2/$10）

**但**：新分词器让同一段输入比 4.6 多产生约 30% 的 token，等价于实际价格上涨 30%。Simon 用他的 Claude Token Counter 工具测了一组大文档确认了这一点。

🔗 https://simonwillison.net/2026/Jun/30/claude-sonnet-5/#atom-everything

### 8. shot-scraper 1.10 + shot-scraper video：让 AI agent 自己录视频 demo

**来源**：Simon Willison｜2026-06-30

shot-scraper 1.10 的主打新功能是 `shot-scraper video` 子命令：接收一个 `storyboard.yml`，定义一组要跑的操作序列，用 Playwright 把整个流程录成视频。

这是 Simon 长期主张的延伸——**AI agent 完成任务后，应该能自己产出可视化的演示**，而不是只丢一段日志或截图。他给出的 demo 是用 shot-scraper 录了一段 Datasette "从粘贴的 CSV/TSV/JSON 创建新表"功能的演示视频。

stoyboard.yml 的设计很极简：场景序列、点击/暂停/输入动作、视口大小、等待选择器——足够驱动任何 Web 流程。

🔗 https://simonwillison.net/2026/Jun/30/shot-scraper-video/#atom-everything

### 9. Giles Thomas：从零写 LLM 训练循环（34a · JAX 版）

**来源**：Giles' blog｜2026-06-30

Giles 用了大半年时间、以 Sebastian Raschka 的《Build a Large Language Model (from Scratch)》为蓝本做了一份"自学科"，最后的毕业任务是：**不参考原书、不参考自己之前写的 PyTorch 代码**、用 JAX 从零训练出一个 GPT-2 small 水平（3.2B token 基础模型）的 LLM。
