### 1. Claude Mac 应用为什么这么烂？因为 Electron 就是他家造的

**来源**: Daring Fireball · John Gruber · 2026-07-03 21:24

Anthropic 的 Claude Mac 桌面应用从 2024 年 10 月发布以来一直被吐槽是 Electron 粗制滥造的产物。John Gruber 终于找到了"罪魁祸首"——Felix Rieseberg。

Rieseberg 目前是 Anthropic 负责 Claude Cowork 和 Claude Code Desktop 的工程负责人，此前主导了 Claude Mac 和 Windows 应用的开发。而他不仅仅是"用过 Electron"——他是 Electron 项目的三位管理委员会成员之一，可以说是 Electron 的核心缔造者之一。

更妙的是，在加入 Anthropic 之前，Rieseberg 曾在 Notion 担任桌面团队工程负责人（Notion 的 Mac 应用是出了名的 518MB Electron 巨怪），更早在 Slack 负责桌面应用团队。Gruber 的比喻绝了：

> "发现一个人——他家是做锤子生意的——先后指挥了泰坦尼克号、驾驶了兴登堡飞艇、然后做了 Amelia Earhart 的空中交通管制员。"

Gruber 还指出，Notion 在去年的 WWDC 上被 Apple 点名表扬，正在迁移到 SwiftUI。而这恰好发生在 Rieseberg 离开 Notion 加入 Anthropic 一年之后。讽刺意味拉满。

### 2. Simon Willison: 让 Fable 自己做判断

**来源**: Simon Willison's Weblog · 2026-07-03 18:51

Simon Willison 分享了一个来自 Claude Code 团队的实用技巧：**别事无巨细地指挥 Fable（Claude 5），让它用自己的判断力。**

比如测试策略——与其告诉 Fable "大功能才写自动化测试，小改动不用"，不如直接说"由你判断什么时候需要写测试"。

更有趣的是 Jesse Vincent 的省钱招数：让 Fable 把小任务委派给更便宜的模型（Sonnet/Haiku）在 subagent 中执行，Fable 自己只负责判断、审查和综合。Claude Code 会把这个偏好保存为 memory 文件，后续自动遵循。

Willison 表示效果很好——大量工作照常完成，但 Fable 的 token 消耗速度明显放缓。

**启示**: AI 编程的最佳实践正在从"精确指令"转向"设定目标 + 授予判断权"。

🔗 https://simonwillison.net/2026/Jul/3/judgement/#atom-everything

### 5. 螺旋蝇的衰落与复兴——一场被遗忘的公共卫生战争

**来源**: Construction Physics · Brian Potter · 2026-07-03 12:02

今年 6 月 3 日，德州 La Pryor 的一头三周大的小牛身上发现了螺旋蝇（screwworm）——这种食肉寄生虫自 1980 年代以来首次重返美国。

这是一篇精彩的历史叙事：

- **螺旋蝇的恐怖**: 在活组织中产卵，幼虫钻入血肉进食， untreated 则致命。1935 年德州就有 120 万例感染、18 万头牲畜死亡。1930-40 年代德州 60-80% 的白尾鹿死于螺旋蝇。
- **消灭方案**: "不育雄性技术"——释放数百万不育雄蝇，雌蝇只交配一次，逐步消灭种群。USDA 花了几十年在美国、墨西哥、中美洲消灭了螺旋蝇。
- **屏障失守**: 自 2000 年代初，美巴联合组织 COPEG 在巴拿马达连隘口维持屏障，每周释放不育雄蝇。约 2023 年屏障失效，螺旋蝇一路北上，如今抵达美国。

**教训**: 我们已经忘记了螺旋蝇曾经是多么可怕的灾难，于是放松了警惕。

🔗 https://www.construction-physics.com/p/the-fall-and-rise-of-screwworm
