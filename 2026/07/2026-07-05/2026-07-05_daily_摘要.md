## 主题一：AI 编程工具与模型演化

### 🔧 sqlite-utils 4.0rc2: Claude Fable 主笔的稳定版冲刺

**核心事件**：Simon Willison 用 Anthropic 在 Max 订阅上仅剩几天的 **Claude Fable** 模型，把 sqlite-utils 推到 4.0 稳定版前夕。这次协作覆盖 **37 个 prompt / 34 个 commit / +1,321 -190 行 / 30 个文件**。

**Fable 挖出的"release blocker"**：
- 最严重：`Table.delete_where()` 没有 `atomic()` 包裹，导致连接永远处于 `in_transaction=True`，后续 `atomic()` 都走 savepoint 分支、从不 commit
- 复现脚本：插入 3 行 → `delete_where` → 关闭 → 重开后**所有数据丢失**（包括无关表 u）
- 此外还有 4 个被 Fable 标为 release blocker 的问题

**作者流程亮点**：Simon 在 Half Moon Bay 独立日游行时用 iPhone 上的 Claude Code for web 逐步指导；Fable 单轮 10–15 分钟，他隔一段时间 cue 一下下一步。

**最终成果**：新的 transaction 模型现在每个写方法（`insert/upsert/update/delete/delete_where/transform/create_table/create_index/enable_fts`）默认都有自己的事务、调用返回前就 commit：

```python
db.table("news").insert({"headline": "Dog wins award"})
# 立刻持久化，无需 commit()
```

### ⚠️ 更好的模型，糟糕的工具调用

**核心发现**：Armin Ronacher 在做 Pi 编程 agent 时发现一个反直觉现象——**新版 Claude 模型在自己专属工具上的调用准确率反而下降**。

**具体表现**：
- Opus 4.8、Sonnet 5 都会在调用 Pi 的 edit 工具时**瞎编 `edits[]` 数组里的字段**
- 编辑内容本身往往是对的，但 schema 不匹配导致 Pi 拒绝并要求重试
- Haiku 等小模型、Claude 旧版本都不会这样

**Armin 的假说**：Anthropic 很可能通过 RL 专门训练新版模型去对齐 Claude Code 内置的 edit 工具。代价是**所有第三方 harness（Pi、Cline 等）的自定义工具被殃及**。

**生态分裂加剧**：
- Claude 的 edit 走 "search and replace"
- OpenAI 的 Codex 走 `apply_patch`
- 模型被专门训练去用各自的"主工具"
- 第三方 harness 该不该**同时实现多套 edit 工具**，看用户选哪个模型给最优性能？

## 主题三：极限编码技巧

### 🗺️ 500 字节的可信世界地图

**奇思**：Iwo Kadziela（Codex 协助）用 **445 字节**的 deflate 压缩数据 + 一段 JS 在浏览器里渲染出一张"看得过去"的 ASCII 世界地图。

**关键 Trick**：通过 `fetch('data:;base64,...')` 拿数据，pipe 通过 `DecompressionStream('deflate-raw')`，再吐到 `<pre>`：

```js
fetch('data:;base64,1ZpLsgIxCEXnrM...==').then(
  r => r.body.pipeThrough(new DecompressionStream('deflate-raw'))
).then(
  s => new Response(s).text()
).then(
  t => b.innerHTML = '<pre style=font-size:.65vw>' + t )
```

把 deflate 直接 fork 给浏览器原生 API 完成，没人这么玩过。

### 🔳 一码两吃：1D 条形码 + 2D QR 码合体

**Terence Eden 尝试**：把 UPC 一维条码嵌进 QR 二维码的"内部图案区"，近距离扫码枪读 UPC、退一步读 QR 整图。

**理由与代价**：
- UPC 加了 padding，扫描器不一定都吃
- QR 用高纠错率，余量充足，本来就能塞图（他 2010 年的发现）
- **代价**：嵌入的 1D 图样破坏 QR 纠错 → QR 变脆弱

**自评**："实用价值不大，但值得把玩。"

**评论区梗**："用食指盖住 QR 拿 Meal Deal 党"——消费者随手一遮就让自助扫码机识别错位的成功率飙升。
