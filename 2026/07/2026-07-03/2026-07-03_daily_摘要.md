**来源**：Daring Fireball 转引 Ookla｜2026-07-02 22:30

### 7. DSPy 优化 Datasette Agent 的 SQL 系统提示

**来源**：Simon Willison's Weblog｜2026-07-02 18:25

Simon 在 AIE 大会上听了 DSPy 的 keynote 后，立刻让 Claude Fable 5 异步跑一个研究任务：使用 DSPy 评估并改进 Datasette Agent 中执行只读 SQL 查询的核心系统提示。

**Fable 的关键发现**：当前 schema 列表只给出表名，而"如果你已经有信息就不要调用 describe_table"的提示反而导致模型猜测列名（page_count、o.order_id、first_name），触发错误重试循环。

**改进方向**：要么在 schema 列表里直接包含列名，要么软化"不要调用 describe_table"的建议。

这印证了 DSPy 的价值——不需要重写 prompt，通过结构化评估自动找到优化点。
