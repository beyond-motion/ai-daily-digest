## 主题一：开发者工具 · sqlite-utils 4.0 — 三年磨一剑的迁移系统

### 🛠️ Simon Willison 发布 sqlite-utils 4.0：数据库模式迁移、嵌套事务、复合外键

**核心事件**：Simon Willison 发布 **sqlite-utils 4.0**——该项目的第 124 次发布，也是自 2020 年 11 月 3.0 以来的首次大版本跳跃。

**三大特性**：
1. **数据库模式迁移**：定义一系列数据库变更，自动追踪哪些已执行、哪些待执行。基于 Python 装饰器 `@migrations()` 编写，支持 `table.transform()` 方法（创建临时表→复制数据→替换原表）
2. **嵌套事务**：新增 `db.atomic()` 方法
3. **复合外键**支持

**迁移追踪机制**：自动创建 `_sqlite_migrations` 表记录已执行的迁移函数。支持 CLI（`uvx sqlite-utils migrate data.db migrations.py`）和 Python API（`migrations.apply(db)`）两种方式。

**历史渊源**：Willison 在 2008 年首届 DjangoCon 上曾与 Andrew Godwin、Russ Keith-Magee 一起展示过相互竞争的 Django 迁移方案。他坦言最终采用了更简单的设计——不支持自动回滚，因为"在 SQLite 项目中，回滚最简单的方法就是在执行迁移前复制一份数据库文件"。

**sqlite-migrate 退役**：同日发布的 sqlite-migrate 0.2 是一个兼容垫片，将老用户引导至 sqlite-utils 4.0。三年独立的 beta 项目正式合入主线。

### 🧩 附：github-code Web Component

Willison 还用 **GPT-5.5** 构建了一个实验性 Web Component `<github-code>`，可以嵌入 GitHub 仓库的代码片段。它将 GitHub URL 转换为 raw.githubusercontent.com 链接，fetch 后展示指定行范围（带行号，暂无语法高亮）。典型的"AI 辅助快速原型"案例。

**意义**：sqlite-utils 4.0 代表了开源工具的一种成熟路径——先在独立包中验证概念（sqlite-migrate beta），在真实项目中使用三年确认设计可靠，然后合并进核心库。不急躁、不冒进。

**意义**：这不是技术批评，而是**文化与金融批评**。Zitron 代表了一种正在扩大的声音——不是反对 AI 技术本身，而是反对围绕它的泡沫叙事、资本循环和准宗教狂热。与昨天 Terence Eden 的"AI 疲劳"一脉相承，但火力全面升级。

## 主题五：计算史 · Windows 95 的"猜"与 Atari 的"袜子国王"

### 🔧 Windows 95 如何判断一个程序是不是安装程序？靠猜。

Raymond Chen（The Old New Thing）揭示 Windows 95 的安装程序检测机制：**纯靠名字中的魔法词**。

| 魔法词 | 备注 |
|--------|------|
| setup | |
| install | 与 inst 冗余 |
| inst | |
| imposta | 意大利语? |
| ayarla | 土耳其语? |
| felrak | 匈牙利语? |

**有趣细节**："install" 条目与 "inst" 冗余——任何包含 "install" 的名字也包含 "inst"。推测是 "install" 先加入，后来发现很多程序叫 "xxx inst"，就加了 "inst" 但忘了删掉 "install"。

如果程序名没匹配，Windows 95 还会检查可执行文件路径中是否包含 "Setup"。多媒体驱动通过 INF 文件安装后，系统会做一次实时文件检查——"大概是多媒体团队发现很多驱动在 INF 文件里覆盖系统 DLL"。

### 🕹️ Ray Kassar — 从纺织品高管到 Atari CEO 的兴衰史

Dave Farquhar 回顾 Atari 前 CEO Ray Kassar（1928-2017）的职业生涯。

**关键节点**：
- 在 Burlington Industries（纺织品）工作 26 年后被 Warner Bros 挖到 Atari
- 1978 年 Warner 解雇 Atari 创始人 Nolan Bushnell 后提拔 Kassar 为 CEO
- 将 Atari 文化从工程驱动转为商业驱动
- 称程序员为"神经质的 Prima Donna"，被员工称为"袜子国王"和"毛巾沙皇"
- 四名程序员要求版税被拒——Kassar 说"你对那个游戏的重要性不比流水线上组装它的人高"——这四人辞职创办 **Activision**，成为史上第一家第三方游戏开发商
- 1982 年 Yars' Revenge 的设计师将 "Ray Kassar" 倒序拼写得到 "Yar" 和 "Razak"
- 1982 年底 Kassar 在财报发布前 23 分钟卖出 5000 股 Warner 股票，SEC 指控内幕交易

**意义**：Kassar 的故事是一个经典警示——当一个公司的成功完全依赖于创意人才时，用工业化管理方式对待他们会怎样？答案是：核心人才流失、创新管道断裂、公司从巅峰跌落。
