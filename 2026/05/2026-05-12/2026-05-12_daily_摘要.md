### James Shore: AI 编码工具必须降低维护成本

**来源**: Simon Willison's Weblog  
**链接**: https://simonwillison.net/2026/May/11/james-shore/#atom-everything  
**时间**: 2026-05-11 19:48

James Shore 提出核心观点：AI 编码工具必须降低维护成本，数学逻辑才能成立：

> 你写代码的速度现在快了一倍？希望你的维护成本也减半了。三倍效率？维护成本必须降到三分之一。否则你就完蛋了。

公式：**产出 × 维护成本**。如果双倍产出但维护成本不变，总成本翻倍；如果双倍产出但维护成本翻倍，总成本变成 4 倍。只有 LLM 降低维护成本，且降低幅度恰好是产出增速的倒数，整个等式才成立。

### Simon Willison: 在脚本 shebang 行中使用 LLM

**来源**: Simon Willison's Weblog  
**链接**: https://simonwillison.net/2026/May/11/llm-shebang/#atom-everything  
**时间**: 2026-05-11 18:48

Simon 发现了一个酷炫技巧：直接在脚本 shebang 中使用 LLM。

```bash
#!/usr/bin/env -S llm -f
Generate an SVG of a pelican riding a bicycle
```

甚至可以调用工具：

```bash
#!/usr/bin/env -S llm -T llm_time -f
Write a haiku that mentions the exact current time
```

或直接执行 YAML 模板定义的 Python 函数工具。

### macOS 26 Tahoe UI 问题根源分析

**来源**: Daring Fireball  
**链接**: https://www.bloomberg.com/news/newsletters/2026-05-10/apple-plans-macos-27-design-changes-latest-on-ios-27-visionos-safari-wwdc-26-mozuaz9m  
**时间**: 2026-05-11 17:55

Mark Gurman 分析 Liquid Glass 在 macOS 上表现不如移动端的原因：
- Liquid Glass 针对 OLED 显示屏设计，现有 Mac 多为 LCD
- MacBook Air 最新设计语言沿用自 2022，MacBook Pro/iMac 来自 2021
- LCD 无法有效呈现半透明、阴影和玻璃效果

**Gruber 的犀利反驳**：把 UI 差归咎于显示器，就像把食物难吃归咎于餐盘太高级——这是 UI 本身的问题。
